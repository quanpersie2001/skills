# Raw Dev Note Entry Template

Use this template when appending one new raw learning entry inside `dev-notes/raws/YYYYMMDD.md`.

**Insert into:** `dev-notes/raws/YYYYMMDD.md` under `## Entries`

Rules for filling this in:
- One entry = one concrete learning.
- Prefer reusable insight over transcript or task log.
- Keep the scope narrow enough that future-you can skim it quickly.
- If the user intent is ambiguous, clarify before filling this in.
- Do not invent fields or omit required ones.
- `status` starts as `raw` and changes only during distillation.

---

## Template

```markdown
### Note: <kebab-case-slug>

**Status:** raw
**Created:** <ISO-8601>
**Summary:** <one-sentence summary of the learning>
**Topic hints:** [hint-1, hint-2]
**Distilled into:** []

#### What I learned

<2-4 sentences describing the reusable insight, heuristic, or reframe.>

#### Why it matters

<1-3 sentences explaining how this changes future coding, debugging, review, or AI collaboration behavior.>

#### Evidence from this session

- <specific observation, experiment, or comparison from the chat/session>
- <second item only if it materially supports the learning>

#### Reuse hint

<one imperative sentence future-you can apply quickly>
```
